CORE REQUIREMENT
---------------------------------------------------

- globalPlayerId
- cross-tournament linking
- historical auction tracking

---------------------------------------------------
REQUIRED LOGGING SYSTEM
---------------------------------------------------

Create backend logging/events architecture for:

1. Bid Events
2. Team Interest Events
3. Player Auction Events
4. Team Strategy Events
5. Timer Interaction Events

Logs should be stored historically and never overwritten.

---------------------------------------------------
BID EVENT LOGGING
---------------------------------------------------

For every bid capture:

- timestamp
- tournamentId
- playerId
- globalPlayerId (future-ready)
- teamId
- bidAmount
- previousBidAmount
- bidIncrement
- bidSequenceNumber
- millisecondsSinceLastBid
- timerRemainingAtBid
- whether bid was manual or live
- whether team became leading bidder

---------------------------------------------------
PLAYER AUCTION LOGGING
---------------------------------------------------

For every player auction capture:

- auction start timestamp
- auction end timestamp
- sold/unsold/deferred
- final amount
- total bids received
- number of interested teams
- highest competing teams
- total auction duration
- average time between bids
- category
- role
- sport type
- player attributes snapshot

---------------------------------------------------
TEAM INTEREST LOGGING
---------------------------------------------------

Track behavioral interest patterns:

- which categories teams frequently bid on
- how aggressively teams bid
- teams that repeatedly target similar player types
- average purse usage behavior
- preferred player roles
- retention patterns
