BIDWAR — OPERATOR PANEL UX RESTRUCTURE
(IMPORTANT: UI/UX redesign only. Do NOT break auction engine, timers, APIs, websocket events, hotkeys, live sync, LED sync or bidding logic.)

CURRENT PROBLEM:
The Operator Panel is visually powerful but mentally exhausting for a first-time operator.

Issues:
- Too much information visible simultaneously
- No visual hierarchy
- Critical actions mixed with secondary actions
- Duplicate information repeated in multiple places
- Important controls don’t stand out properly
- New operator cannot instantly understand:
  - where to focus
  - what to click next
  - what is currently happening
  - what is safe vs dangerous

The panel should feel like:
“Auction Control Desk”
NOT
“A dense admin dashboard”

====================================================
PRIMARY UX GOAL
====================================================

During live auction:
Operator should think ONLY about:
1. Current Player
2. Current Bid
3. Timer
4. Main Action Buttons

Everything else should become secondary or collapsible.

====================================================
CRITICAL RULE
====================================================

DO NOT REMOVE FEATURES.

Instead:
- reorganize
- prioritize
- simplify visibility
- reduce duplication
- improve hierarchy

Advanced operators should still have access to all controls.

====================================================
NEW UI STRUCTURE
====================================================

The screen should be divided into:

----------------------------------------------------
ZONE 1 — LIVE AUCTION CORE (CENTER)
----------------------------------------------------

This is the HEART of the system.

Only most important things should dominate here.

CENTER PRIORITY:
1. Player Name
2. Current Bid
3. Timer
4. Start/Pause bidding
5. Sold / Unsold

These should visually dominate the screen.

====================================================
REMOVE VISUAL NOISE
====================================================

Currently:
- too many outlines
- too many tiny labels
- too many repeated categories
- too many repeated player details
- too many small sections competing equally

Reduce:
- borders
- repeated text
- tiny labels
- unnecessary headings

Use spacing instead of excessive boxes.

====================================================
CURRENT PLAYER CARD RESTRUCTURE
====================================================

Current player card contains too many small scattered details.

NEW FORMAT:

----------------------------------------------------
PLAYER NAME (BIG)
Category + Role (small)
Base Price
Age + City (secondary)
----------------------------------------------------

REMOVE DUPLICATES.

If category already visible once,
don’t repeat it elsewhere.

Player image should become optional/minimal.

====================================================
CURRENT BID AREA
====================================================

This should become the BIGGEST visual focus.

CURRENT BID should dominate center.

Example hierarchy:

₹2,00,000
Current Bid

Smaller below:
Base Price ₹1,50,000

Current raise info should be subtle.

====================================================
MAIN ACTION BUTTONS
====================================================

Current issue:
Too many equal-weight buttons.

NEW PRIORITY:

PRIMARY ACTIONS:
- START BIDDING
- SOLD

SECONDARY:
- UNSOLD
- DEFER

TERTIARY:
- MANUAL

====================================================
BUTTON COLOR LOGIC
====================================================

Green:
Safe / progression actions

Yellow:
Attention / transition

Red:
Destructive/final

Blue/Grey:
Utility actions

====================================================
NEXT PLAYER BUTTON
====================================================

This should NOT dominate more than bidding controls.

Currently oversized.

Reduce visual weight slightly.

====================================================
TIMER UX
====================================================

Current timer visibility is weak.

Timer should become:
- bigger
- cleaner
- always visible
- easier to understand under pressure

Use:
large circular countdown OR prominent top timer bar.

Operator should instantly know:
- running
- paused
- ended

without reading text.

NEW STRUCTURE:

Auction Status

Current Auction Phase

====================================================
REMOVE DUPLICATE INFORMATION
====================================================

Currently repeated:
- category
- player count
- labels
- team info

Audit entire panel and remove duplicate text.

Rule:
One information = one place only.

PLAYER QUEUE PANEL

- equal emphasis everywhere

TEAM NAME

pure black everywhere.

====================================================
ACTIVE STATES
====================================================

- what is active

====================================================
PRESSURE MODE UX
====================================================

IMPORTANT:

This panel is used during LIVE AUCTION PRESSURE.

Operator cognition reduces under stress.

Therefore:
- fewer decisions visible simultaneously
- fewer competing colors
- fewer tiny texts
- larger interaction targets
- clear action hierarchy

The UI should support:
FAST CONFIDENT ACTIONS.

====================================================
FINAL UX PHILOSOPHY
====================================================

The operator panel should feel:
calm,
focused,
and pressure-resistant.

At any moment the operator should instantly know:
- who is being auctioned
- current amount
- timer state
- next action

WITHOUT searching the screen.
