BIDWAR UX/UI & ONBOARDING RESTRUCTURE PLAN

IMPORTANT:
This is a UI/UX onboarding and flow improvement task ONLY.

DO NOT break:
- existing APIs
- database schema
- auction logic
- socket events
- timer logic
- operator workflows
- live sync
- owner panel functionality
- OBS/LED integrations
- authentication flow
- local mode/export architecture
- tournament IDs
- existing routes
- role permissions
- reports logic
- current business logic

This is NOT a backend rewrite.

Goal:
Make BIDWAR extremely beginner-friendly for first-time organisers while preserving ALL existing functionality and logic.

---------------------------------------------------
CORE PRINCIPLE
---------------------------------------------------

Every screen should answer:
1. What is this?
2. Why does it matter?
3. What should I do next?

The product should feel like:
“Guided tournament setup”
NOT
“Complex auction software”

---------------------------------------------------
MAIN OBJECTIVE
---------------------------------------------------

A first-time organiser should be able to:
- Login
- Create tournament
- Configure rules
- Add teams
- Add players
- Start auction

WITHOUT external training.

---------------------------------------------------
CRITICAL RULES
---------------------------------------------------

1. DO NOT REMOVE EXISTING FEATURES
Even advanced settings must remain available.

2. DO NOT CHANGE API CONTRACTS
Frontend adaptation only unless absolutely required.

3. DO NOT BREAK OLD TOURNAMENTS
Backward compatibility required.

4. KEEP EXISTING DATA STRUCTURE
Only improve presentation, onboarding, defaults, and flow.

5. ALL CURRENT FEATURES MUST STILL WORK
Even if hidden behind “Advanced Settings”.

---------------------------------------------------
PHASE 1 — ONBOARDING EXPERIENCE
---------------------------------------------------

After Google Login:

Instead of directly opening the dashboard,
show:

WELCOME SCREEN

Heading:
“Welcome to BIDWAR”

Subtext:
“Let’s setup your first auction tournament.”

Primary CTA:
[ Start Tournament Setup ]

Secondary CTA:
[ Explore Demo ]

---------------------------------------------------
PHASE 2 — GUIDED SETUP FLOW
---------------------------------------------------

Create a setup wizard with progress tracking.

STEPS:

Step 1 → Tournament Details
Step 2 → Auction Rules
Step 3 → Teams
Step 4 → Players
Step 5 → Ready for Live Auction

Each step should show:
- completion state
- lock/unlock state
- next action

Example:
✅ Tournament Setup
🔒 Teams Locked Until Rules Complete

---------------------------------------------------
PHASE 3 — TOURNAMENT SETTINGS UX
---------------------------------------------------

Current settings modal is overwhelming.

Split into:

BEGINNER MODE (default)
ADVANCED SETTINGS (expandable)

---------------------------------------------------
BEGINNER MODE FIELDS
---------------------------------------------------

MANDATORY ONLY:

- Tournament Name
- Sport
- Total Points Per Team
- Minimum Player Value
- Bid Increment
- First Bid Timer
- Subsequent Bid Timer
- Minimum Squad Size

Hide advanced complexity initially.

---------------------------------------------------
TOOLTIPS / HELP SYSTEM
---------------------------------------------------

Every important field should have:
ⓘ info icon

Hover/click explanation.

Examples:

TOTAL POINTS PER TEAM
“Each team gets this total purse for buying players.”

MINIMUM PLAYER VALUE
“No player can start below this amount.”

BID INCREMENT
“Every new bid increases by this amount.”

FIRST BID TIMER
“Time available before the first bid.”

SUBSEQUENT TIMER
“Timer reset after every new bid.”

MINIMUM SQUAD SIZE
“Minimum players every team must purchase.”

---------------------------------------------------
BID TIER LOGIC IMPROVEMENT
---------------------------------------------------

Current 3-layer tier system is confusing for beginners.

NEW DEFAULT:
Single increment system only.

Example:
Increment = ₹10,000

Advanced users can enable:
[ Enable Advanced Bid Tiers ]

Then show:
- Tier 1
- Tier 2
- Tier 3

IMPORTANT:
Do NOT remove existing tier logic.
Only simplify initial experience.

---------------------------------------------------
SMART DEFAULTS
---------------------------------------------------

Pre-fill sensible defaults:

Bid Increment = 10000
First Bid Timer = 30
Subsequent Timer = 15
Minimum Squad = 11

Organiser can edit later.

---------------------------------------------------
SAVE BUTTON IMPROVEMENT
---------------------------------------------------

Replace:
“Save Changes”

This improves flow clarity.

---------------------------------------------------
PHASE 4 — LOGICAL FLOW ENFORCEMENT
---------------------------------------------------

Prevent confusing navigation.

RULES:

Before Teams:
Mandatory tournament setup must be complete.

Before Players:
Minimum 2 teams required.

Before Live Auction:
Require:
- 2 teams
- at least 1 player
- required rules completed

Show friendly instructional messages instead of errors.

Example:
“Please create at least 2 teams before adding players.”

PHASE 16 — SAFETY & STABILITY
---------------------------------------------------
